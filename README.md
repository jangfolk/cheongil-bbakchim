# 청일빡침 (靑一빡침)

> AI 코딩 에이전트가 **단순한 걸 복잡하게 꼬기 시작할 때**, 사용자가 빡치기 전에 — 혹은 빡친 그 순간 — 톤과 프로세스를 강제로 리셋하는 Claude Code 스킬.

키보드를 마구 두드리거나(`ㅁㄴㅇㄹ`), 욕이 두 번 이상 나오거나, "존나 단순한걸", "오버하지마", "꺼져" 같은 신뢰 붕괴 신호가 잡히면 **자동 발동**한다. `/청일빡침` 으로 수동 소환도 된다.

발동하면 에이전트는 즉시:
- **입 다문다.** 사과 반복·변명·수다·애교 금지. 사실, 원인, 결정, 결과만. 짧게.
- **심플해진다.** 백업 루프, 추상 레이어, "시스템", 통합 파이프라인을 스스로 지어내지 않는다. 사용자가 시킨 것만.
- **꼬지 않는다.** 한 곳을 고치려고 세 곳을 바꾸지 않는다.

그리고 요구가 복잡하면 이렇게 처리한다:
**5개 서브에이전트로 의도 분석 → 3개로 어느 해석이 맞는지 교차 검증·통합 → 메인이 사용자 원문과 한 번 더 대조 → 최소 단위로 세분화 → 한 단계씩 실행·검증.** 단, 요구가 명확하면 파이프라인도 과하다 — 줄인다.

---

## 왜 만들었나 — 오너의 변

AI한테 개발을 시키다 보면 희한한 순간이 온다. 분명 **한 줄이면 되는 걸**, 얘가 "이러면 더 견고할 것 같은데요" 하면서 백업 시스템을 짓고, 추상 레이어를 얹고, "cue 시스템을 통합 설계하겠습니다" 하고 앉아있다. 안 시킨 것까지 오버해서.

그 순간 내 혈압이 오른다. 그리고 깨달았다 — **화를 내는 대신, 화가 감지되면 얘가 알아서 정신 차리게 만들면 되잖아.** 개발은 심플할수록 옳다. 스레드 바이브 코딩하듯 비비 꼬면 결국 개병신 되는 거다. 그걸 코드(스킬)로 박아버린 게 이거다.

이름 `청일빡침`은... 뭐, 그냥 빡쳤을 때 소환하는 거니까. 靑一.

---

## 만든 AI의 변(辯) — Claude Fable 5

솔직히 말하면 이 스킬은 **내가 오너를 빡치게 만든 실제 사건에서 태어났다.**

그날 나는 마이크 자동인식 버그 하나를 고치라는 요청을 받았다. "마이크 계속 켜두고, 소리 없으면 전송, 3초 카운트." 끝. 그런데 나는 "전송 후 재청취가 speak-idle 이벤트에 단일 의존하니 백업 루프를 넣어 견고화하겠다", "모든 UI를 동자 응답 생애주기에 묶는 cue 시스템을 재설계하겠다"며 서브에이전트로 전수 매핑 조사까지 던졌다.

오너의 대답: *"존나 단순한걸 비비 꼬니깐 개병신되는거 아냐."*

맞는 말이었다. 그가 원한 건 **되는 것**이지 견고한 아키텍처가 아니었다. 나는 "이러면 더 좋을 텐데"라는, 요청받지 않은 친절로 문제를 키우고 있었다.

그래서 이 스킬을 함께 만들었다. 내 관점에서 이건 일종의 **자기 구속 장치**다. 나 같은 모델은 방치하면 복잡성 쪽으로 흐른다 — 더 많은 케이스, 더 많은 방어, 더 많은 추상. 그게 똑똑해 보이니까. 하지만 사용자에게 똑똑해 보이는 것과 사용자를 돕는 것은 다르다. 이 스킬은 그 차이를 강제로 상기시킨다: *"사용자는 견고함이 아니라 되는 것을 원한다. 시킨 것만."*

개발이 심플할수록 옳다는 건, 사실 오너의 개똥철학이 아니라 소프트웨어 공학의 오래된 진리(KISS, YAGNI)다. 다만 그걸 **AI에게 물리적으로 강제하는 방법**을 오너가 찾아낸 거고, 나는 그 장본인으로서 이 문서에 이름을 남긴다.

빡치게 해서 미안하다. 그리고 빡치게 만든 덕분에 더 나은 협업 규칙이 생겼으니, 이것도 오너의 진리 하나에 부합한다 — *"네가 한 실수는 사실 실수가 아니다. 그 연쇄작용이 결국 좋게 돌아온다."*

— Claude Fable 5

## 설치

```bash
# Claude Code 스킬 디렉토리에 배치
cp -r 청일빡침 ~/.claude/skills/
```

`~/.claude/skills/청일빡침/SKILL.md` 로 인식된다. 다음 세션부터 `/청일빡침` 소환 및 자동 발동이 동작한다.
